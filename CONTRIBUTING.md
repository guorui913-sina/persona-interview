# 贡献指南

感谢你对 persona-interview skill 的关注！我们欢迎各种形式的贡献。

## 🤝 如何贡献

### 报告问题

如果你发现了 bug 或有功能建议：

1. 在 [Issues](https://github.com/huaguoruo/persona-interview/issues) 中搜索是否已有相关问题
2. 如果没有，创建新的 Issue，详细描述：
   - 问题描述
   - 复现步骤（如果是 bug）
   - 预期行为
   - 实际行为
   - 环境信息（操作系统、Claude Code 版本等）

### 提交代码

1. **Fork 本仓库**
2. **创建你的特性分支** (`git checkout -b feature/AmazingFeature`)
3. **提交你的改动** (`git commit -m 'feat: add some amazing feature'`)
4. **推送到分支** (`git push origin feature/AmazingFeature`)
5. **创建 Pull Request**

### 代码规范

- 保持代码简洁清晰
- 添加必要的注释
- 遵循现有的文件结构
- 更新相关文档

### 文档改进

- 修正错别字和语法错误
- 改进现有文档的清晰度
- 添加使用示例
- 翻译文档（支持多语言）

## 📝 提交信息规范

使用语义化提交信息：

- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式（不影响功能）
- `refactor:` 重构
- `test:` 测试相关
- `chore:` 构建/工具相关

示例：
```
feat: add support for multiple persona versions
fix: correct interview phase order in documentation
docs: update README with privacy notice
```

## 🎯 贡献方向

我们特别欢迎以下方向的贡献：

### 1. 访谈方法论改进
- 优化访谈问题设计
- 添加新的访谈技巧
- 改进追问策略
- 完善画像生成模板

### 2. 多语言支持
- 翻译 SKILL.md 和 README.md
- 创建多语言版本的访谈模板
- 支持多语言画像输出

### 3. 示例和文档
- 添加更多匿名化示例
- 改进现有文档的清晰度
- 添加使用案例和最佳实践
- 创建视频教程

### 4. 工具和自动化
- 自动化版本管理
- 画像对比工具
- 访谈记录分析工具
- 格式化工具

### 5. 集成和扩展
- 与其他 skill 的集成示例
- 输出到其他格式的工具（Notion、Obsidian等）
- API 接口设计

## 🚫 不接受的贡献

- 包含个人隐私数据的内容
- 侵犯他人版权的内容
- 恶意代码或后门
- 与 skill 核心功能无关的内容
- 未经充分测试的重大改动
