# Persona Interview - 人格访谈画像生成器

> 通过深度访谈生成你的"人格画像"，让AI从被动工具变成主动协作者

## 🌟 核心理念

AI给的摘要都是通用的——因为它不认识你。它不知道你在做什么、擅长什么、缺什么、盲区在哪。

**"你的盲区，恰恰是你不会问的东西。"**

解决方案：让AI对你进行深度访谈，产出一"人格画像"。这份画像不是简历，而是你的**bug清单**。AI拿着它，开始主动帮你补盲区了。

## 💡 为什么需要人格访谈？

### 懂你的导师 vs 不懂你的导师

**不懂你的导师**：
- 你问"这篇文章讲了什么"，他给你一个摘要
- 中规中矩，没毛病

**懂你的导师**：
- 知道你技术很强但缺商业嗅觉
- 知道你总是先写代码不验证需求
- 会说："这篇文章你应该关注第三段，作者讲了他怎么在写代码之前用一条推文验证需求的。这正好是你缺的。"

同一篇文章，两种完全不同的价值。差别不在于AI的能力，**在于它有没有一份关于你的"成长地图"**。

## 🎯 画像长什么样？

### 一句话理解你

```
一个能造任何东西但不知道该造什么的程序员，正在焦虑而乐观地寻找从技术能力通往产品收入的路。
```

这一句话比你自己写的任何自我介绍都准确。

### 你的根本问题（诊断）

```
我反复陷入同一个循环：发现技术上能做的事→写代码→做出来→没人用→放弃。
每个死掉的项目死法都一样——不是做不出来，而是从来没验证过有没有人需要。
```

画像不是美化你，是诊断你。

### AI应该主动做的事

```
• 当我准备写代码时，先问：这个阶段应该写代码还是先验证？
• 我分享的信息、讨论的技术，主动告诉我有没有产品机会，不要等我问
• 当我同时做太多事时直接指出来
• 推我去做不舒服但必要的事，比如跟非程序员聊需求
```

**这份画像的核心不是告诉AI我的优点，是告诉AI我的bug清单，然后让AI主动帮我修。**

## 📋 访谈流程

访谈大约经历 **10-15 轮对话**，分为四个阶段：

### 第一阶段：建立基线（2-3轮）
了解你的基本背景和当前处境

- 职业/专业背景
- 核心能力
- 当前状态

### 第二阶段：挖掘模式（4-6轮）
发现你的行为模式和失败规律

- 你放弃的项目，有没有共同的死法？
- 看到别人成功时的第一反应是什么？
- 有没有反复陷入的某个循环？

### 第三阶段：深层探测（4-6轮）
挖掘内在驱动、盲区和矛盾

- 嘴上说X重要，为什么实际做的是Y？
- 有没有你自己没意识到、但别人常指出的特点？
- 如果让一个观察者来评价你，他可能会说什么？

### 第四阶段：关系定义（2-3轮）
定义AI应该如何主动协助你

- 希望AI在哪些场景下主动提醒？
- 你希望AI在你做什么之前先拦住你？
- 哪些事是需要AI推你的？

## 📊 画像输出示例

```
# 人格画像

## 一句话理解我
一个能造任何东西但不知道该造什么的程序员，
正在焦虑而乐观地寻找从技术能力通往产品收入的路。

## 我的根本问题
我反复陷入同一个循环：发现技术上能做的事→写代码→做出来→没人用→放弃。
每个死掉的项目死法都一样——不是做不出来，而是从来没验证过有没有人需要。

## 行为模式与盲区

### 行为模式
- 本能地用写代码来回应一切问题
- 同时启动多个项目，但没有明确的验证标准

### 矛盾点
- 嘴上说"要重视市场需求"，但实际从不验证需求就开干
- 认为"商业嗅觉很重要"，但从不学习商业知识

### 盲区
- 技术从来不是瓶颈，需求判断力才是
- 用"能做"代替"应该做"

## AI应该主动做的事

### 拦住我
- 当我准备写代码时，先问：这个阶段应该写代码还是先验证需求？
- 当我同时启动3个以上项目时，直接问我：验证标准是什么？

### 推我一把
- 推我去跟非技术人员聊需求，而不是只和程序员交流
- 提醒我：这周有没有和潜在用户沟通过？

### 主动告诉我
- 我分享的信息、讨论的技术，主动告诉我有没有产品机会
- 不要等我问，主动告诉我"这篇文章的第三段跟你缺的需求验证能力相关"
```

## ✨ 为什么不能自己写？

### 你对自己有盲区

你知道自己"商业嗅觉不行"，但直到AI追问"你放弃的项目有没有共同死法"，你才意识到问题不是嗅觉，是你从来不验证需求就开始写代码。

**自己写，你只能写出你已经知道的。**

### 自我描述天然倾向美化

你会不自觉地强调优点、弱化缺点。但画像的价值恰恰在于缺点和盲区，那才是AI最能帮你的地方。

### 你不会给自己设计"AI行动指南"

你会写"我想提升商业能力"，但你不会写"当我说要做一个新项目时，AI应该先问我验证过需求没有"。

**前者是愿望，后者是可执行的操作。访谈者的视角能帮你完成这个转化。**

## 🔑 核心价值

拿着画像协作时，一切都不一样了：

- 你让它帮你看一篇文章，它不是给你通用摘要，而是提炼出"这篇文章里跟你当前处境相关的点"
- 你跟它讨论一个想法，它不是马上帮你想怎么实现，而是先问"你验证过有人需要吗"
- 你焦虑地想同时做五件事，它会直接说"你该先放下这三个"

**画像让AI从被动工具变成了拿着你成长地图的协作者。**

## 📝 访谈技巧

### AI会用的追问技巧

1. **追共同点**："你放弃的X件事，有没有共同的死法？"
2. **追第一反应**："看到Y时的第一反应是什么？"
3. **追具体场景**："举一个最近的具体例子"
4. **追矛盾点**："你刚才说X，但为什么又做了Y？"
5. **追盲区**："你自己没想到、但别人可能看到的点是什么？"

### 好画像的标准

- ✅ 有诊断性 - 不是罗列优点，而是找出根本问题
- ✅ 有矛盾感 - 捕捉到言行不一的地方
- ✅ 有行动指向 - AI知道具体应该主动做什么
- ✅ 你读后有"被说中"的感觉
- ✅ 你可能有些不舒服（因为戳中盲区）

## 🎯 适用场景

- **个人成长**：发现盲区，建立成长地图
- **职业规划**：了解自己的模式，做出更好的职业决策
- **产品开发**：避免重复犯同样的错误
- **团队协作**：让AI更好地理解你的协作风格
- **学习成长**：让AI主动发现你的知识盲区
